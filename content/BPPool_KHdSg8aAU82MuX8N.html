  : Callout

Application looks up candidate reviewers in IMPAC II and also allows creation of new Person Records in IMPAC II.

 

Includes FACA compliance in detecting NCAB (and other secondary review committee members), which must not also participate in R29 level review.

 

Also flagging of potential COI based on Institutional Profile File (IPF) for reviewers and PI associated with the same institution.

  : Callout

Unassigned Reviewers get removed from list

  : Callout

COI is typically indicated by potential reviewers, can discussed with SRO, who ultimately flags the COI and COI Type before Assignment.

 

If COI is conformed then the Reviewer is either barred from see the Application or granted a waiver (can see Application but must leave meeting during discussion of the conflicted Application.

  : Callout

Reviewer Recruitment via email.

  : Callout

Reviewers fill out Voter Sheet to Score Application.

 

  • Old Process: Numerical scores from 1-5.
  • New process: Numerical Score from 1 - 9 per several criteria (varies by Application Type/topic)
  • Both processes can have non-numerical scores as well

 

  : Callout

This includes preparation of the Master and Individual Voter Sheets and creating a securely shared password for meeting materials.

  : Callout

Releasing scores changes the Application status from Pending IRG to either Pending Council or Not Discussed; calculates priority scores and assigns percentiles, if applicable; updates IRG Recommendation based on Score; notifies PI of the review outcome; and makes the priority Score available in eRA Commons.

  : Callout

Who sets the codes for Application:

 

  • A: Animal Subject Code assigned to the Application
  • H: Human Subject Code assigned to the Application
  • G: Gender Clinical Study Code assigned to the Application
  • M: Minority Clinical Study Code assigned to the Application
  • C: Child Clinical Study Code assigned to the Application
  • T: Clinical Trials Code assigned to the Application

 

  : Callout

Summary Statement (SS) is an official review outcome document that Peer Review users create Summary Statement for each reviewed Application. Summary Statement include reviewer evaluations, scores, and related comments. The Application and meeting information contained in the IMPAC II database are also part of the Summary Statement. The bodies of these Summary Statement are prepared on local computers at the conclusion of the review meeting and are uploaded to the IMPAC II database for inclusion in the final Summary Statement preparation.

  : Callout

Summary Statement cannot be edited within the eRA Application module but in an external text processor followed by reimporting into Peer Review

  : Callout

Summary Statement Release is managed by changing access privileges in Peer Review.

  : Callout

Summary Statement are distributed as zipped files per release date and scored vs unscored. to DEA email and POs are notified in separate email.

  : Callout

SRO can also apply Scientific Terms to Application. How is this different from RCDC? What is the significance?

  : Callout

Peer Review provides the ability for Peer Review Officers (SRO) and Extramural Support Assistants (ESA) to edit the text of abstracts and Public Health Relevance (PHR) statements.

  : Callout

To preserve the anonymity of reviewers in CSR and IC SEP (SEP) meetings, the SEP aggregate roster is created, combining reviewers in SEP meetings. Each CSR and IC SEP panel can have one or more SEP aggregated rosters per Council round and IRG Cluster.

  : Callout

Application Related Materials can be burnt on a passcode encrypted CD to be sent to the Reviewer.

CDs are collected back at time of meeting.
