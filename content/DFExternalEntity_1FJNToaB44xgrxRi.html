1.    Purpose

The purpose of this document is to present the functionality of the I2E Grant Info Web Service.  It describes a RESTful API based interface to retrieve customized data from the I2E Grant Info web service by other web Application or systems.

2.    Overview

The I2E Grant Info Web Service retrieves a list of selected Grant data records from the I2E database based on the predefined resource configuration.  This service is useful to an Application that needs to retrieve Grant data and does not need direct access to I2E database.

3.    Operations

 

 

Title

Retrieve a list of Grant data records from the given resource in JSON format

URL Example

https://I2E.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.json?limit=100 

 

where “funded_Grant” is the name of the selected Grant data resource

and limit is optional maximum number of Grant records to retrieve

 

Method

GET

URL

Params

Required: 

None

 

Optional:

limit=[integer] – number of Grant data records to retrieve.  Default value is 10, defined by the resource configuration. 

example:

https://I2E.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.json?limit=5 

 

By default, the system cannot return more than 999 records.  Please, contact the support team to adjust this limit.

 

offset=[integer] – Index of R29 record to retrieve. Default value is 0.

example:

https://I2E.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.json?limit=5&offset=2 

 

sort=[+|-][string] – The property to order the returned records by.  Default is unsorted.  The property can be preceded by the sign. “+” means ascending sort order, “-“ means descending order.  When sign is absent the default sort order is ascending.

example:

https://I2E.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.json?limit=5&sort=-update_date 

 

<field name>=<field search value> – Search criteria for the given field of the data records.  User can have multiple criteria for several fields in the data record

example:

https://I2E.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.json?limit=5&ACTIVITY_CODE=R13&Grant_type_code=5 

 

Response

Format

JSON

Success

Response

Response Code: 200

Content Example:

{“total”:25000, “page”:0,"size":2,"data":[{"serial_number":69856, "project_title":"Clinical Trials”}, {"serial_number":12345,"project_title":" Cancer Research”}],”error”:””}

 

Error

Response

Response Code: 200

Content Example: (in case if user does not have authority for the resource or the resource is not found)

{“total”:0, “page”:0,"size":0,"data":null,”error”:”You are not authorized for the Grant_view resource”}

4.    Authentication

The I2E Grant Info Web Service uses NCI Active Directory (AD) authentication.  The I2E Grant Info Web Service supports LDAP authentication over SSL to secure the Account credentials.  The proper example of URL to the web service will be:

https://user:password@I2E.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.xml

If the client Application passes wrong credentials, the “401:  Unauthorized” response code will raise the exception on the client side.

 

5.    Authorization

The Web Services authorization access is provided on resource level.  The web service administrator sets the access rights to a requested Grant Info resource for a given NIH AD Account.

7.    Request for Grant Resource

The get access to the existing Grant resource client must provide the NIH AD Account to the web service administrator.

 

To create and to get access to customized Grant resource client must provide the following information:

 

NIH Active Directory Account(s)

Single or multiple accounts to have an authority to access the requested Grant resource.

Description (optional)

Description of the desired Grant resource(s).  For example, the request resource should retrieve only funded Grant information.

The Grant properties to retrieve (optional)

Description of Grant properties to retrieve if client wants to customize the default list of fields (See Appendix 4 for the list of default fields)

Maximum number of records (optional)

The optional maximum number of records to retrieve with a single request.  This limitation can be used for performance purposes.

 

 

 


Appendix

1.    Accessing Service using a web browser

Using web browser to access the service is very simple.  From a browser, such as IE or Firefox, enter in the URL of the web service as following:

 

https://I2E.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.xml

 

https:// I2E.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.json

 

where “funded_Grant” is actual name of the resource.

For web service URLs to non-production tiers see  Domain Hosts and Environments section.

 

When the browser prompts for user name and password, enter the NIH AD Account's username and password.

 

A document will be returned and displayed in the browser.

2.    Accessing Service from Java CAA Example

To access Grant Info RESTful Web Service, you need Apache HHTPClient library.  To parse JSON response, you can use a library like Apache Jackson.  To parse XML response, you can use the built JAXB package in the JDK.

 

R29, define your Grant bean (your fields should correspond to your view configuration:

 

public class Grant {
    private int serial_number;
    private String institution_name;

    public Grant() {}
   
    public int getSerial_number() { return serial_number; }
    public void setSerial_number(int serial_number) {
        this.serial_number = serial_number;
    }

    public String getInstitution_name() { return institution_name; }
    public void setInstitution_name(String institution_name) {
        this.institution_name = institution_name;
    }

 

    @Override
    public String toString() {
        return "Grant{" + "serial_number=" + serial_number +
             ", institution_name='" + institution_name + '}';
    }

}

 

Next, create Grant List Response bean as following:

 

import javax.xml.bind.annotation.XmlAccessType;
import javax.xml.bind.annotation.XmlAccessorType;
import javax.xml.bind.annotation.XmlElement;
import javax.xml.bind.annotation.XmlRootElement;
import java.util.List;

@XmlRootElement(name = "grantListXml")
@XmlAccessorType(XmlAccessType.FIELD)
public class GrantList {
    private int total;
    private int page;
    private int size;
    @XmlElement(name = "Grant")
    List<Grant> data;
    private String error;

    public GrantList() {}

    public int getTotal() { return total; }
    public void setTotal(int total) { this.total = total; }

    public int getPage() { return page; }
    public void setPage(int page) { this.page = page; }

    public int getSize() { return size; }
    public void setSize(int size) { this.size = size; }

    public List<Grant> getData() { return data; }
    public void setData(List<Grant> data) { this.data = data; }

    public String getError() { return error; }
    public void setError(String error) { this.error = error; }

 

    @Override
    public String toString() {
        StringBuilder sb = new StringBuilder();
        sb.append("total=").append(total).append("\n");
        sb.append("page=").append(page).append("\n");
        sb.append("size=").append(size).append("\n");
        if (data != null) {
            sb.append("data[length=").append(data.size()).append("]\n");
            for (Grant g : data) {
                sb.append("    ").append(g.toString()).append("\n");
            }
        }
        else {
            sb.append("data[length=0]\n");

        }
        sb.append("error=").append(error).append("\n");
        return sb.toString();
    }
}

 

To access and parse XML representation of the response use the code below.  Change the user name, password, host, and funded_Grant accordingly.  The “limit=2” parameter is optional.

 

import org.apache.http.HttpEntity;
import org.apache.http.client.methods.CloseableHttpResponse;
import org.apache.http.client.methods.HttpGet;
import org.apache.http.impl.client.CloseableHttpClient;
import org.apache.http.impl.client.HttpClients;

import javax.xml.bind.JAXBContext;
import javax.xml.bind.Unmarshaller;
import java.io.InputStream;

public class MainXml {

    public static void main(String[] args) throws Exception {
        CloseableHttpClient httpclient = HttpClients.createDefault();
        try {
            HttpGet httpGet = new HttpGet("https://user:password@I2E.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.xml?limit=2");
            httpGet.setHeader("Accept", "Application/xml");
            CloseableHttpResponse response = httpclient.execute(httpGet);
            HttpEntity entity = response.getEntity();
            InputStream is = entity.getContent();

            JAXBContext context = JAXBContext.newInstance(GrantList.class);
            Unmarshaller unmarshaller = context.createUnmarshaller();
            GrantList result = (GrantList) unmarshaller.unmarshal(is);
            System.out.println(result.toString());
        } finally {
            httpclient.close();
        }
    }
}

 

To access and parse a JSON representation of the response, use the code below.  Make necessary changes as described in the paragraph above.

 

import com.fasterxml.jackson.databind.ObjectMapper;
import org.apache.http.HttpEntity;
import org.apache.http.client.methods.CloseableHttpResponse;
import org.apache.http.client.methods.HttpGet;
import org.apache.http.impl.client.CloseableHttpClient;
import org.apache.http.impl.client.HttpClients;
import java.io.InputStream;

public class MainJson {

    public static void main(String[] args) throws Exception {
        CloseableHttpClient httpclient = HttpClients.createDefault();
        try {
            HttpGet httpGet = new HttpGet("https://user:password@I2E.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.json?limit=2");
            httpGet.setHeader("Accept", "Application/json");
            CloseableHttpResponse response = httpclient.execute(httpGet);
            HttpEntity entity = response.getEntity();
            InputStream is = entity.getContent();

            ObjectMapper mapper = new ObjectMapper();
            GrantList result = mapper.readValue(is, GrantList.class);
            System.out.println(result.toString());
        } finally {
            httpclient.close();
        }
    }
}

3.    Accessing Service from Ruby CAA

You can access the service from Ruby CAA using rest_client gem.  The following example accesses Grant Info resource and parses JSON format response.

 

require 'rest-client'
require 'json'

class Launcher
  def initialize
  end

  def run
    json = RestClient.get 'https://user:password@host.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.json?limit=2', {:accept => :json}
    resp = JSON.parse(json);
  end
end

 

l = Launcher.new
l.run

 

To access Grant Info resource with XML response and parse XML formatted response use the following example.  There are many XML parser gems.  The crack gem parses XML response into a hash map.

 

require 'rest-client'
require 'crack'

class Launcher
  def initialize
  end

  def run
    response = RestClient.get 'https://user:password@host.NCI.NIH.gov/i2egrantinfo/services/Grant/funded_Grant.json?limit=2', {:accept => :json}
    resp = CRACK::JSON.parse(response);
  end
end

l = Launcher.new
l.run

4.    Description of Generic View provided by I2E Grant Info Web Service

The Service provides the default “funded_Grant” Grant View.  The client can request a custom view specifying which fields should be included in the custom view, and what type of Grant should be retrieved – funded only, unfunded only, or both.  The Service returns 10 records if the parameter limit is not defined.  The Service cannot return more than 999 records per request.
