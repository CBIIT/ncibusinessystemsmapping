Description

Coding, as defined in this document, includes four distinct processes. The first and third of these processes are primarily associated with extramural awards, while the second and fourth apply both to intramural projects and extramural awards.

 

  • Initial coding upon the receipt of an application, for the purpose of referral to the proper Division/program officer and to assist in the review process. Other codes (e.g., Program Class Codes (PCCs), which include CA and PD codes) are also applied at this stage.
  • Cross-organizational scientific coding of extramural and intramural awards. This includes the CSO, SITE, and SIC codes. RAEB applies scientific codes to extramural awards, while intramural PIs apply scientific codes to their own projects.
  • Division-level coding of extramural awards, using codes defined by a particular NCI organization, but not used across the Institute.
  • Congressionally mandated, NIH-wide automated (with the exception of a few categories) coding for Research, Condition, and Disease Categorization (RCDC), which categorizes all NIH research projects.

 

Each of these coding processes is carried out through distinct systems. Initial coding occurs through the I2E Referral and PD Assignment modules. Cross-organizational scientific coding occurs through FLARE (for extramural), through ARS (for CCR intramural), and through the DCEG Intramural System (for DCEG intramural). Division- level coding uses the Program Coding Module or specific other systems (e.g., PMA for DCCPS). NIH-wide categorization occurs through the (mostly) automated RCDC process/system.

For more information on RCDC coding process, see: https://report.nih.gov/rcdc/process.aspx
