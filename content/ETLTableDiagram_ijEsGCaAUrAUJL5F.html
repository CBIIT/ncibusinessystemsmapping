The NED Portal/NED (NED) - is a centrally coordinated, electronic directory containing identity, contact, and organizational information for all members of the NIH workforce and non-NIH staff using NIH services and facilities. NEDWeb is the custom data content and process management Application used by NIH administrative personnel for adding new people to NED, updating existing records, transferring people between ICs, and deactivating people when they leave NIH. NEDWeb is also used for authorizing NIH "services" such as ID badges, Library privileges, Active Directory accounts, Exchange mailboxes, VPN Remote access, parking permits, and listing in the NIH Telephone and Services Directory. NEDWeb provides the following features and capabilities:

  • Work flow

  • Scheduled tasks (e.g., the ability to schedule registrations, updates, etc., to occur on a future date)

  • Event driven e-mail Notification

  • Ability to save tasks for completion at a later time

  • Reporting capabilities

  • Business process integration with NIH and HHS systems (e.g., HHS Identity Management System, Background Investigation System) to support the HSPD-12 Personal Identity Verification process

  • User Administration

This document provides detailed information on the use of NEDWeb and is intended to serve as a reference guide for system users.  Please direct comments or suggestions to ned@mail.NIH.gov.

DCB Portfolio Analysis Dashboard -  DCP Portfolio Analysis Dashboard PPT and DCP Portfolio Analysis Dashboard Demo 

  • Overview 

  • Late 2018, DCP created an Executive Dashboard as a one-stop shop to provide a means to identify, track, and RePORT details on DCP-funded clinical research projects and operation budgets.  The online resource will provide managers with administrative and scientific data and performance metrics to aid in data analysis, reporting, and decision-making at the branch- and division-wide level.   

  • The SITE is currently being tested. However, based on user testing, this automation of the division's key performance indicators will provide the ability to perform effective oversight of DCP's portfolio, and continually identify operational challenges and proactively apply corrective measures when needed.  

  • Goals 

  • See timely data DCP from major CAA and Research Groups. 

  • Save time/resources with automated reports and visualizations 

  • Access DCP data from anywhere and on any device 

  • Get alerts via email when KPIs need attention 

  • Functions/Features 

  • Custom views – different access/views based on role and responsibility 

  • Ability to drill into details – create interactive tools and reports for analysis 

  • All-in-one – unify data form multiple sources into a single repository 

  • Up-to-data information – current data in real-time or near real-time data  

  • Promote knowledge transfer – empower staff with  visualization, charts, table, maps that are easy to share and educate  staff  

  • Users 

  • 15 users selected across DCP 

  • Roles/only able to access what is required for your job 

  • Budget User – can access the Budget page 

  • Cancer Care Delivery User – can access the CCD trials page 

  • Clinical Trial User – can access all trials pages and view the trials reports 

  • Grant User – can access all the Grant section 

  • Dashboard Admin – all the above access privileges  

  • Dashboard Superuser – all SITE permissions including adding and editing users/roles/graphs 

  • Access 

  • Access must be approved by Protocol Information Office (PIO) Management 

  • There’s no way to request an Account, only Dashboard Superusers can create new accounts 

  • Must use PIV card (Users are authenticated against NIH iTrust) 

  • Will provide clear guidance for everyone – mobile iron/PIV card 

  • Hosted on contractor’s SITE 

  • Tech Platform 

  • Front-end: HTML & Bootstrap for layout and the nvd3 JavaScript charting library & jquery for dynamic content. 

  • Back-end: Django Application framework and a Postgres database. 

Tech Stack Status 

  • Currently using Django 1.9 but will be upgrading over the next few months to Django 2.x. 

System development, operations and maintenance Contract

The NIH authoritative sources from which Beacon HR downloads data are HRDB, FPS, NED, and NVISTAS. For each of these sources, only IC specific information is downloaded.

Account Management Services (AMS)  provides self service and delegated management of various accounts associated with NIH people.
