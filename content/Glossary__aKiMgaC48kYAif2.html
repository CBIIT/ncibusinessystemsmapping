Program Funding Requests is an I2E application for NCI Extramural program staff to create and submit initial pay (exception), supplement, skip requests that require approvals within NCI. It also supports the creation of RFA/PA funding plans that will be submitted for SPL approval. Application provides electronic workflow, enforcement of funding levels of authority policy, and automated approval for funding requests and plans.

 

I2E Program tool used to document funding actions that fall outside of the normal process of paying grants in rank order (Paylist).

 

Program Funding Request (PFR) is a module that allows the NCI Extramural Program staff to create and submit Program Funding Requests (Initial Pay, Supplement), Skip Requests and Funding Plans that is routed for electronic approvals based on the NCI Levels of Authority (LOA) policy.

Supports compliance with the NIH audit requirements by providing appropriate electronic approval documentation for awarding grant applications.

 

Core Business Functions:

  • Allows the NCI Extramural program staff to create and submit Initial Pay, Supplement, Skip, and Funding Plan requests for automated approvals by the Scientific, Funds, and Grants Management officials.
  • Enforces compliance with the NCI funding policies by determining the Scientific Levels of Authority (LOA) for requests based on the following criteria:
    • Cost Center/Budget Mechanism of the Grant application
    • Request Type
    • Funding Source
    • Program recommended dollar amount
    • IMPAC II Grant Status
    • Grant Priority Score and Percentile
    • Fundable Score Range and DOC exception selection parameters defined in the Paylist Manager system
  • Support RFA/PA Funding Plans:
    • Allows the Program staff to create a standard Funding Plan for a single or combined RFA/PA spanning one or more applicable NCAB dates and provide funding details for each application selected for funding.
    • Determines the applications that will be skipped and not considered for funding in a Funding Plan, by allowing the Program staff select the applications for funding consideration and define the Fundable Score Range.
    • Facilitates SPL presentation process, by supporting the generation of the Funding Plan word document templates with pre-populated data that could be used by the Program staff to create the Scientific Rationale and justification for skipped applications.
  • Provides the ability to upload justification and supporting documents needed for the approval of the request
  • Supports System Integration:
  • Compares the fundable score range defined for the Cost Center/Budget Mechanism in the Paylist Manager system to the Grant application score or percentile to determine the appropriate request types that can be selected for a specific grant application.
    • Integrates with the Grants Portfolio Management and Tracking System (GPMATS) – Upon the completion of the PFR, the system automatically creates Action Requests in GPMATS for OGA to track award/post award actions
    • Supports the Pre-appl Concept for OEFIA: PFR Integrates with OEFIA’s FAM module to do one or more of the following: Display the active and projected CAN in the OEFIA Funds approval page, Set/Update the Official Pay Code, Assign CAN number based on the selection of pre-appl or roll-up option, Support TCS calculation, Record the funding source usage based on the request type.
  • Creates Funding Package of fully approved PFRs and Funding Plans for automated retrieval by OGA’s eGrants system
  • Allows the users to track the status of request.
  • Enables automated email notifications
  • Built-in search capabilities: Performs default search based on the logged-in user’s role
  • Report Generation: Generates PDF or Excel report on the search results

 

Primary Users:

  • Create PFR: Program Directors and Other Program Staff
  • Submit PFR: Requesting Program Directors (OR) Program Directors associated with the same Cancer Activity as the Requesting Program Director
  • Scientific Approval (determined by the system):
    • Program Directors
    • Division/Center Directors
    • NCI Director
    • Scientific Program Leaders (SPL) Committee Designee
  • Funds Certifier: ARC Funds Approvers and OEFIA Financial Analysts
  • Grants Management Approval: OGA Chiefs

 

Size of Customer Base:

  • NCI Program Staff
    • NCI Program Directors – 300
    • Other Program Staff – 130
  • Division/Center Directors (includes Acting/Deputy Division Directors) - 19
  • NCI Director or Designee - 1
  • Scientific Program Leaders (SPL) Committee Designee - 1
  • ARC Funds Approvers - 13
  • OEFIA Financial Analysts - 8
  • Grants Management Approvers - 4
