User Info API consists of a set of REST web services which pull identity information from the Virtual Directory Server (VDS) and NED. VDS provides an integrated view of NED and Active Directory (AD) identity information for users. Additionally, NED tracks historical change information (of NED information only, AD information changes are not tracked) and exposes an API which returns user account changes within a given time range. Further integration of property information hosted in nVision/Sunflower and additional user information from NCI Frederick databases is currently under development.

  ChangesByIC : REST Resource

This API returns records that have changed in NED within a given time range. Note that there is no change tracking feature for AD information. Therefore, periodic full updates from VDS are necessary in order to avoid getting out of sync with AD changes.

  • Request Body format: JSON or XML formatted object containing the Icore Site and a time range from which changed records are requested.
  • Default Response Format: XML
  • Resuest headers: Authorization (for basic authentication), Accept (for requested response format. optional - defaults to application/xml)

User Records Filtered by IC
  • URL: https://<host>/api/vds/users/ic/{IC_ACRONYM}
  • Request Type: GET
  • Result Format: JSON or XML formatted array of user objects containing the users' NED and AD information (refer to VDS_InternalView_SchemaProd_20180831.xlsx for list of exposed fields).
  • Default response format: XML
  • Request headers: Authorization (for basic authentication), Accept (for requested response format. optional - defaults to application/xml)
  • This service makes an LDAP call to VDS to get all users falling under the specified IC.
